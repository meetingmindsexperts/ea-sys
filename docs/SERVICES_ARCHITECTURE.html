🛠 8. What's next

  • Phase 2c — registration-service (shipped April 23). The two callers were already aligned by Phase 0, which turned the planned deferral into a safe consolidation opportunity. Landed earlier than the original plan suggested. Backward compat preserved end-to-end (verified by independent review-agent with 45/45 checklist items).
  • Phase 3 — External REST API (next forcing function). A public REST API for third-party integrators is on the near-term roadmap. New endpoints back onto the existing four services. Any domain operation the API exposes that doesn't yet have a service gets its own extraction at that point — shape informed by a real third caller, not guessed.
  • Phase 4 — Opportunistic (ongoing). Single-caller routes extract only when touching for a feature reason. No dedicated time.
Why 2c landed early: Originally deferred because premature extraction would have locked in a service shape before the external-API caller's requirements were known. But Phase 0 had already aligned the two existing callers exactly, meaning the service shape was already observed from two real implementations. Extracting became a low-risk consolidation rather than a speculation. Experienced senior-review answer: "Extract when the shape is known from real callers. We had two, aligned. Waiting for a third to redundantly confirm the shape would be ceremony."
