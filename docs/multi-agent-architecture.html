Executive Summary

The Problem

Today's AI agent is a single chatbot on a standalone page. Staff must leave their work, navigate to the agent page, describe their context, and hope the agent picks the right action from 12 generic tools. It doesn't understand domain nuance, can't connect to external services like Google Calendar or Teams, and there's no audit trail of what it does.

The Solution

Replace the single agent with a supervisor + 6 specialized sub-agents, each embedded directly where staff work:

  • Registration Agent on the Registrations page — ticketing, payments, attendees
  • Speaker Agent on the Speakers page — speaker management, invitations, imports
  • Schedule Agent on the Schedule page — sessions, tracks, calendar sync
  • Abstracts Agent on the Abstracts page — submissions, reviews, scoring, reviewer assignments
  • Accommodation Agent on the Accommodation page — hotels, room types, room allocations
  • Communications Agent on the Communications page — email drafting, templates, campaigns
  • Supervisor in the sidebar — always accessible, routes cross-domain requests, posts to Teams

Each agent has multi-step skills (e.g., "send reminders to all unpaid registrations"), connects to external services (Google Calendar, Microsoft Teams, EventsAir), and every action is logged to a full audit trail with human approval gates for destructive operations.

Architecture

The 7 Agents

🤖

Supervisor Agent — Always visible in sidebar

The orchestrator. Routes complex, cross-domain requests to the right sub-agent(s), aggregates results, and notifies the team via Microsoft Teams.

  • Delegates to Registration, Speaker, and Schedule agents
  • Can invoke multiple sub-agents for multi-domain tasks (e.g., "set up the full event")
  • Direct access to event info for quick status checks
  • MCP: Microsoft Teams — posts to configured channels on task completion
📋

Registration Agent — Registrations page

Handles all registration, ticketing, and payment operations. Accessible via a floating AI button on the Registrations page.

  • Tools: List/create registrations, list/create ticket types
  • Skills: Send payment reminders, generate event summary, bulk confirm paid registrations
  • Direct API: Stripe (payment status, refunds), Email (Brevo/SendGrid)
🎤

Speaker Agent — Speakers page

Manages speakers and speaker communications. Accessible via a floating AI button on the Speakers page.

  • Tools: List/create speakers, send bulk emails
  • Skills: Send speaker invitations, import speakers from registrations
  • Direct API: EventsAir (import speaker data), Email (Brevo/SendGrid)
📅

Schedule Agent — Schedule page

Manages sessions, tracks, and the event schedule. Accessible via a floating AI button on the Schedule page.

  • Tools: List/create sessions, list/create tracks, list speakers
  • Skills: Auto-schedule sessions, send schedule updates to attendees
  • MCP: Google Calendar — create/update calendar events when sessions are created
📝

Abstracts Agent — Abstracts page

Manages the full abstract lifecycle — submissions, reviews, scoring, and status transitions. Accessible via a floating AI button on the Abstracts page.

  • Tools: List abstracts, update abstract status, assign reviewers, list review criteria, list themes
  • Skills: Bulk assign reviewers by specialty match, send status notifications (accepted/rejected/revision), generate review statistics report, flag overdue reviews
  • Direct API: Email (Brevo/SendGrid) for status notification emails
🏨

Accommodation Agent — Accommodation page

Manages hotel bookings, room types, and room allocations for event attendees. Accessible via a floating AI button on the Accommodation page.

  • Tools: List hotels, list room types, create/update accommodation bookings, check availability
  • Skills: Auto-assign rooms by registration type, generate rooming list report, flag overbooking conflicts
✉️

Communications Agent — Communications page

Drafts emails, manages templates, and sends targeted campaigns. Accessible via a floating AI button on the Communications page.

  • Tools: List/create email templates, send bulk email, preview email with branding, list recipients by segment
  • Skills: Draft event announcement email, create targeted campaign by registration status/type, generate post-event thank you emails
  • Direct API: Brevo/SendGrid for delivery with branding wrapper and CSS inlining

Human Approval Gates

Agents are powerful — but irreversible actions at scale require human confirmation before execution. The system enforces approval gates on high-impact operations.

⚠️ How it works: When an agent is about to perform a bulk or destructive action, it pauses and presents a confirmation prompt to the user: "I'm about to send payment reminders to 147 unpaid registrations. Proceed?" The agent waits for explicit user approval before executing.
ActionGate TypeWhy
Send bulk email (any type)Confirmation + recipient countEmails are irreversible; wrong blast damages reputation
Bulk confirm registrationsConfirmation + countStatus change triggers downstream effects (badges, access)
Create > 5 records in one requestConfirmation + summaryPrevents runaway creation from ambiguous prompts
Post to Microsoft TeamsPreview message before sendExternal-facing; wrong message visible to entire channel
Send schedule update to attendeesConfirmation + recipient countMass communication about schedule changes
✅ Safe by default: Read-only operations (listing, searching, generating reports) execute immediately with no confirmation. Only write operations that affect multiple records or reach external systems require approval.

Not everyone should have the same agent capabilities. Access is controlled by the user's existing EA-SYS role.

🔒 How it's enforced: The existing denyReviewer() API guard and role whitelist on the agent API route are reused. Members can ask questions and view data through agents, but all write tools return "Forbidden" at the API level. No client-side-only enforcement.

External Integrations

Agents connect to external services through two mechanisms, each suited to different integration patterns:

MCP Connectors (Model Context Protocol) Open standard for AI tool integration. The agent discovers available tools dynamically from the MCP server. Best for: services where the AI needs to choose which action to take.
Direct API Integrations Custom tool executors that call REST/GraphQL APIs directly. Already built into EA-SYS for existing services. Best for: established integrations with known, fixed operations.
ServiceTypeAgentWhat It DoesStatus
Google Calendar MCP Schedule Create/update calendar events when sessions are created or changed Phase 1
Microsoft Teams MCP Supervisor Post notifications to configured channels (via Graph API) Phase 1
EventsAir Direct API Registration, Speaker Import events, contacts, and speaker data (GraphQL + OAuth2). Already built. ✅ Existing
Stripe Direct API Registration Check payment status, process refunds, generate checkout sessions. Already built. ✅ Existing
Brevo / SendGrid Direct API All agents Send transactional and bulk emails. Already built with provider auto-detection. ✅ Existing
Supabase Storage Direct API All agents Upload and serve media files (photos, documents). Already built. ✅ Existing
🔌 Extensible by design: New MCP connectors (e.g., Zoom, Mailchimp, BigMarker, n8n) can be added by registering an MCP server config on the relevant sub-agent. No code changes to the core agent loop — just a config entry with the server command and auth credentials.

How Agents Learn & Improve

Each sub-agent has a skills file — a markdown document that defines its personality, domain expertise, prompt templates, and learned behaviors. Admins can edit these files to teach agents new patterns without writing code.

Skills File Structure

Each sub-agent reads a skills file at startup that shapes how it thinks and responds. These live in the codebase as simple markdown:

What Goes in a Skills File

📖 Domain Knowledge

Business rules the agent should know: "Early Bird pricing closes 30 days before the event." "Speakers from academia get complimentary registration." "The keynote track is always in the Main Hall."

✉ Prompt Templates

Pre-written email templates for common scenarios: payment reminders, speaker invitations, schedule change notifications. The agent uses these instead of generating from scratch.

🚨 Guardrails & Rules

What the agent should never do: "Never change a CONFIRMED registration to CANCELLED without approval." "Never send more than 500 emails in one batch." "Always include the event name in subject lines."

🔄 Learned Patterns

Patterns discovered from the audit trail: "When users ask 'who hasn't paid', they usually want a list sorted by registration date." "Session conflicts are most common on Day 1 morning slots."

3 Ways Agents Improve Over Time

MethodWho Does ItHow It WorksExample
1. Manual skill edits Admin / Developer Edit the .md skills file directly. Changes take effect on next agent request — no deploy needed (files are read at runtime). Add a new email template for "abstract acceptance" to speaker.md
2. Audit trail analysis Admin / Developer Review the Agent Activity log to find common patterns, frequent failures, or repeated corrections. Codify the fix as a new rule in the skills file. Users keep correcting the agent's session duration from 30min to 45min → add rule: "Default session duration is 45 minutes"
3. Per-event overrides Organizer Store event-specific agent instructions in the Event settings JSON. These are merged with the base skills file at runtime, allowing per-event customization. For a medical conference: "All speaker titles must use 'Dr.' prefix." For a tech event: "Default track names are Frontend, Backend, DevOps."
🧠 Why this matters: Traditional chatbots are static — they behave the same way forever. With skills files, agents accumulate organizational knowledge over time. After 6 months of running events, the registration agent will know your pricing patterns, common attendee questions, and preferred email tone — all captured in a simple markdown file that anyone on the team can read and edit.
🔒 Skills files are version-controlled: Since skills files live in the codebase (src/lib/agent/skills/*.md), every change is tracked in Git with full history. You can see who changed what, when, and roll back if an edit causes problems. Per-event overrides are stored in the database with the same audit trail as other event settings.

Audit Trail & Monitoring

Every Agent Action is Logged

A dedicated AgentAuditLog database table captures every tool execution across all agents, providing full traceability for compliance and troubleshooting:

FieldDescription
AgentWhich agent executed (supervisor, registration, speaker, schedule)
ToolWhich tool or skill was called
InputSanitized input parameters (passwords, tokens, API keys automatically stripped)
OutputResult summary (truncated to 2KB)
SuccessWhether the action succeeded or failed
DurationExecution time in milliseconds
ChainLinks sub-agent actions to supervisor delegations (parent → child tracing)
UserWho triggered the action
TimestampWhen it happened

Viewable by admins at /events/[eventId]/agent-activity with filters by agent, user, date range, and success/failure. Supervisor → sub-agent chains are linked for full end-to-end traceability.
