Dependency Upgrades — Round 1

EA-SYS · Prepared July 23, 2026 · Snapshot taken against ea-sys@0.4.24 on main

Status: PROPOSED — awaiting owner go-ahead. Nothing in this document has been applied. This is the first structured dependency-upgrade round for this project; the format here is the template for future rounds. Re-run npm outdated and npm audit before executing — versions drift weekly.

1 · Principles for a live production platform

  • Minors and patches within our current majors only. These are the versions a fresh npm install would already resolve for a new developer — we are catching the lockfile up, not changing any API contract.
  • Vendor SDK majors are projects, not hygiene. A Stripe or Prisma major changes the contract with the vendor and gets its own planned round with dedicated testing.
  • One revertable commit, one deploy. The whole batch ships as a single commit so the drilled 22-second pinned rollback (docs/ROLLBACK.md) covers all of it at once.
  • Full verification gate before push (tsc · lint · 3,666 unit tests · next build) plus the e2e suite and a post-deploy production smoke (§5).
  • Dormant code paths are not upgraded. Upgrading an SDK we don't actively run (Brevo, SendGrid, Postmark) buys risk and zero value.

5 · Rollout plan

  1. Pre-flight: re-run npm outdated/npm audit (targets may have moved); confirm CI idle (gh run list) and no live event in the next 48h.
  2. Apply: explicit npm update <the §3 list> (not a blanket update — Zoom/next-auth must not move), bump exact-pinned react/react-dom to 19.2.8, pin @zoom/meetingsdk to 6.0.0. Lockfile + package.json in one commit.
  3. Gate (local): npm ci → npx tsc --noEmit → npm run lint → npx vitest run (3,666) → npm run build → npm run test:e2e (stop the dev server first — port 3113 collision).
  4. Deploy: push to main → CI (checks ∥ tests ∥ build → parallel image build → blue-green deploy, ~8 min). Health-gated container swap means a boot-time failure never reaches nginx.
  5. Post-deploy smoke (~10 min):
    • Login + dashboard loads; open one registration detail sheet (Radix), edit → save (RHF/zod)
    • Public register page: welcome + terms render (dompurify), price table, mobile width
    • Template test-send → EmailLog row SENT (SESv2)
    • Badge print one registration (bwip-js/pdfkit) — scan it on the check-in page
    • /worker/health 200 + worker:tick-end lines in /logs (node-cron/tsx)
    • /admin/infra cards populate (AWS clients); Sentry receives a test event
    • Public agenda + a session page render (CSS/Tailwind visual check)
  6. Rollback if needed: IMAGE_TAG=<previous-sha> bash scripts/deploy.sh — drilled, ~22 s, zero downtime (docs/ROLLBACK.md). No DB migrations in this batch, so rollback has no data dimension.
