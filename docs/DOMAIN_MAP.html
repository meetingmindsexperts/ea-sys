EA-SYS Domain Map

The context-reload tree — click any branch to expand. Last updated July 13, 2026 (post accommodation-review remediation; certificates review, pkg 0.4.20). Markdown-free, self-contained, dark-mode aware.
How to use: jump to the domain you're about to touch → open Core logic first (those files are the single source of truth for the domain's rules) → check Gotchas and Review & deferred before assuming a behavior is a bug — it may be a known, deliberately-deferred item. When changing domain logic, walk every entry-point leaf (UI / REST / Public / MCP / Worker): the historical bug class here is two entry points drifting apart.

⚙️Core logic

lib/accommodation-rooms.ts — the room-counter model: holdsRoom() · pure planRoomTransition(prev,next) (the ONE net move; makes double-counting unrepresentable) · guarded releaseRoom() · atomic claimRoom() · releaseRoomForDeletedPerson()

⚠️Gotchas
THE INVARIANT: bookedRooms == count of accommodations on that room type that aren't CANCELLED. Never move it by hand — go through planRoomTransition(). Two blocks each moving the counter is exactly the bug that shipped (H3).
Cascade hazard: Accommodation onDelete: Cascades from Registration AND Speaker. A DB cascade fires NO application code — any new person-delete path MUST call releaseRoomForDeletedPerson() inside its tx or the room is lost forever.
Releases are floored (bookedRooms > 0): negative would make available exceed totalRooms and turn a counting bug into physical overbooking.
Price auto-computed nights × pricePerNight; one accommodation per registration/speaker (unique FKs — though the create refine is "at least one", not "exactly one" — LOW L1).
Guest count is validated on CREATE only — the PUT never re-checks it against room capacity (deferred M2).
