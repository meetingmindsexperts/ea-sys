Zoom Integration — EA-SYS Implementation Guide • April 2026

Complete documentation of the Zoom integration into EA-SYS, enabling meetings, webinars, and webinar series linked to event sessions with a branded attendee landing page.

1. Overview

The Zoom integration is a fully decoupled, optional module that adds live meeting and webinar capabilities to event sessions. It operates at three levels:

LevelControlWhen Off
OrganizationZoom OAuth + SDK credentials in Org SettingsAll Zoom API routes return 404
EventEvent.settings.zoom.enabled toggleNo Zoom UI in session forms
SessionPer-session "Create Zoom Meeting" buttonSession works as before

Video/audio streaming happens entirely on Zoom's infrastructure. EA-SYS stores meeting metadata, generates JWT signatures, and provides a branded landing page for attendees.

React 19 Limitation: The Zoom Meeting SDK (@zoom/meetingsdk) bundles React 18 internally and is incompatible with React 19 (used by Next.js 16). Embedded in-browser meetings are not currently possible. The integration uses a branded landing page that redirects attendees to Zoom's web client instead. The embed component (zoom-embed.tsx) is preserved for future use when Zoom releases a React 19 compatible SDK.

2. Architecture

Organization Settings (DB) └─ settings.zoom: { accountId, clientId, clientSecretEncrypted, ← Server-to-Server OAuth sdkKeyDev, sdkSecretDevEncrypted, ← Meeting SDK (Development) sdkKeyProd, sdkSecretProdEncrypted, ← Meeting SDK (Production) sdkMode: "dev" | "prod" ← Active SDK mode } │ ├─── Server-to-Server OAuth ──→ Zoom REST API │ /v2/meetings, /v2/webinars, /v2/users/me │ Used for: create, update, delete meetings/webinars, manage panelists │ └─── Meeting SDK credentials ──→ JWT Signature Generation Used for: generating attendee join signatures (future: embedding) │ ▼ Public Join API (/api/public/events/[slug]/sessions/[sessionId]/zoom-join) │ ├─ SDK configured → { mode: "sdk", sdkKey, signature, meetingNumber } └─ SDK not configured → { mode: "url", joinUrl } │ ▼ Public Session Page (/e/[slug]/session/[sessionId]) │ └─ Branded landing page with event banner, session details, speakers with photos, and "Join Meeting" button → Opens Zoom web client in new tab

Two Zoom Apps Required

App TypePurposeCredentialsWhere Stored
Server-to-Server OAuth Create/manage meetings & webinars via API Account ID, Client ID, Client Secret Org Settings (encrypted)
General App JWT signatures for attendee join + future embedding Client ID (SDK Key), Client Secret (SDK Secret) Org Settings (Dev + Prod, encrypted)

Credential Security

FieldStorage
accountId, clientIdPlain text
clientSecretEncryptedAES-256-GCM encrypted
sdkKeyDev, sdkKeyProdPlain text (public key, sent to browser)
sdkSecretDevEncrypted, sdkSecretProdEncryptedAES-256-GCM encrypted

Encryption uses NEXTAUTH_SECRET as the key derivation input via SHA-256. Secrets are never returned by the GET endpoint — only hasClientSecret: true/false flags. Pino logger auto-redacts fields containing "secret".

2Server-Side Zoom Module

Created src/lib/zoom/ with five files following the lazy-init singleton pattern:

New Files

  • New src/lib/zoom/types.ts — TypeScript interfaces (credentials with dev/prod SDK fields, meeting params, responses)
  • New src/lib/zoom/client.ts — OAuth token management with in-memory cache + TTL, zoomApiRequest() wrapper with duration logging
  • New src/lib/zoom/meetings.ts — Meeting/webinar/series CRUD, panelist management, all operations logged with apiLogger
  • New src/lib/zoom/signature.ts — Org-aware JWT signature generation (generateZoomSignatureForOrg), picks dev/prod based on sdkMode
  • New src/lib/zoom/index.ts — Re-exports + isZoomConfigured() helper

Key Design Decisions

  • OAuth tokens cached in-memory — Map<orgId, {token, expiresAt}> with 5-minute pre-expiry refresh, cache hits logged at debug level
  • Credentials encrypted with AES-256-GCM — reuses encryptSecret()/decryptSecret() from eventsair-client.ts
  • Dev/Prod SDK credentials — stored per-org with sdkMode toggle; dev for localhost testing, prod for live domain
  • No environment variables needed — all credentials come from org settings in the database
  • Secrets optional on update — if left blank on save, existing encrypted values are preserved

  • Event branding — banner image, organization name, event name
  • Session details — title, date, time, location, capacity, track badge
  • Status badges — Live Now (green pulse), Upcoming, Ended
  • Speakers sidebar — photos (with initials fallback), names, job titles, organizations
  • "Join Meeting" CTA — prominent blue button that opens Zoom web client
  • Meeting details card — type, meeting ID, passcode
  • "Not yet joinable" state — shows when session hasn't started, with countdown info

Data fetched in parallel from two endpoints: /zoom-join (meeting info) and /detail (session + event branding + speakers).

4. Setup Guide

ACreate Server-to-Server OAuth App (for meeting management)

  1. Go to Zoom App Marketplace → Develop → Build App
  2. Choose Server-to-Server OAuth app type → Create
  3. Name it (e.g., "EA-SYS Event Platform")
  4. On the App Credentials tab, copy: Account ID, Client ID, Client Secret
  5. On the Scopes tab, add:
    • meeting:write:meeting:admin, meeting:read:meeting:admin
    • webinar:write:webinar:admin, webinar:read:webinar:admin
    • user:read:user:admin
  6. Click Activate

BCreate General App (for Meeting SDK / embedding)

  1. In Zoom App Marketplace → Develop → Build App
  2. Choose General App type → Create
  3. On the App Credentials tab, copy: Client ID (= SDK Key) and Client Secret (= SDK Secret)
  4. On the Scopes tab, add: meeting:read, meeting:write
  5. On the Embed tab, add allowed domains:
    • localhost (for development)
    • events.meetingmindsgroup.com (for production)
  6. Set OAuth Redirect URL to https://zoom.us (not used — just a required field)
  7. The app has both Development and Production credentials — copy each set separately

CConfigure in EA-SYS

  1. Go to Organization Settings → Integrations tab
  2. In the Zoom Integration card:
    • Server-to-Server OAuth: Account ID, Client ID, Client Secret from Step A
    • Meeting SDK — Development: Dev Client ID + Secret from Step B
    • Meeting SDK — Production: Prod Client ID + Secret from Step B
    • Active SDK Mode: "Development" for testing, "Production" for live
  3. Click Save Credentials → Test Connection
  4. Go to your event → Settings → Zoom tab → Enable Zoom
  5. Go to Schedule → click on a session → click "Create Zoom Meeting"
  6. Use "Start as Host" to begin the meeting, share "Open Embed Page" link with attendees

6. Performance & Capacity

Zoom streaming runs entirely on Zoom's infrastructure. EA-SYS only serves a lightweight JWT signature and branded landing page.

ScenarioServer Impact
2,000 attendees join within 5 min~7 req/s to /zoom-join — single DB read + JWT sign (~2ms each)
Multiple webinars same day, different slotsLoad staggered naturally. Each session has its own endpoint.
After joinZero server load — Zoom runs in the attendee's browser/app.

Performance optimizations implemented:

  • OAuth token cache — in-memory Map<orgId, {token, expiresAt}> with 5-minute pre-expiry refresh. Cache hits logged at debug level.
  • Promise.all — all routes parallelize independent DB queries (event + session + zoom meeting)
  • Prisma select — all queries use select (not include) for minimal data transfer
  • @zoom/meetingsdk in serverExternalPackages — keeps 5MB SDK out of server bundle
  • Dynamic import — zoom-embed.tsx loaded with next/dynamic + ssr: false
  • No N+1 queries — panelists, speakers, and session data fetched in single queries with nested selects
  • Public endpoints stateless — no auth overhead, single DB read + JWT sign per join request

The EC2 t3.large instance handles this comfortably. No upgrade needed for Zoom itself.
