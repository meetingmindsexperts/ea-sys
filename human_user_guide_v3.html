Getting Started
Understanding the system, finding your event, reading the dashboard.
All roles
What the system does

The MMG New System is Meeting Minds Group’s central event management platform. It manages the complete lifecycle of every event — from the first speaker invitation to post-event invoicing. It handles registrations, accommodation, abstract submissions, speaker agreements, communications, and financial tracking in one place.

The system connects directly to your AI agents. When an agent is asked to register a speaker or send payment reminders, it uses the same platform you see in the dashboard. Everything stays in sync.

Conference vs. Webinar — what’s different
ℹ️
Event type affects available features
Webinar events have additional tools: attendance tracking, engagement reports (polls, Q&A), and Zoom/BigMarker integration. Conference events have fuller accommodation and day-of check-in features. See Section 12 for webinar-specific guidance.

Day-of speaker swap

When a speaker cancels on the day, you no longer need to delete and recreate sessions. The system supports atomic speaker swaps that preserve the session ID, topics, and delegate associations.

Go to the session → Speakers panel
Click Remove next to the cancelling speaker
Click Add Speaker → search for the replacement
Assign the same role
Session ID, topics, and programme order are all preserved automatically
✅
Or use the atomic replace
For a full swap of multiple speakers, use Replace All Speakers — define the entire new speaker list for a session in one step with no intermediate state.

Registration & Ticketing
Ticket types, promo codes, registering attendees, statuses, check-in, and walk-ups.
Delegate Support · Event Coordination · Finance
Ticket types explained

Ticket types define attendee categories for your event: Physician, Allied Health Professional, Student, Resident, Member, Complimentary.

⚠️
Ticket pricing is set in the dashboard only
When a ticket type is created it starts at $0 with 3 inactive pricing tiers (Early Bird, Standard, Onsite). To set actual prices go to Settings → Registration Types and activate the relevant tier. AI agents cannot set pricing.
Promo codes
IMG-010
Promo Codes List
Promo codes table showing: code, discount type (% or fixed), usage count, validity dates, linked ticket types, and active/inactive toggle.
⚠️
Deleting a promo code is a soft delete
Deleting sets the code to inactive — it stops working for new registrations but existing registrations that used it are not affected. You can reactivate at any time.
Registering an attendee
IMG-011
Registration Form
Add registration form — fields: Title, First name, Last name, Email, Organisation, Ticket type selector, Status selector, and optional special requirements.
Navigate to Registrations → Add Registration
Fill in: Title, First name, Last name, Email, Organisation
Select a Ticket type
Set Status — CONFIRMED for direct registrations, PENDING for those awaiting payment
Click Register — a QR code and serial ID are generated automatically and a confirmation email is sent

Day-of check-in
IMG-012
Check-in Interface
Day-of check-in screen — QR scanner view on left, manual name/email search on right, with check-in confirmation toast and running checked-in count.

Scan the QR code from the attendee’s confirmation email or badge, or search by name. The system marks them as CHECKED_IN with a timestamp.

Handling cancelled registrations (walk-ups)
Option A — Reinstate First
Update status from CANCELLED to CONFIRMED, then check in normally. Correct if the cancellation was an error.
Option B — Walk-up Override
Use Allow Cancelled Check-in override. Logged in the audit trail. Use for genuine walk-ups where reinstating first is impractical.

Finance & Invoicing
Invoice numbering, types, tracking, chasing unpaid registrations, and refunds.
Finance & Operations · Leadership
How invoice numbers work

Invoice numbers are generated automatically using the event code. The format is: [EVENT_CODE]-INV-001, [EVENT_CODE]-INV-002, etc. The event code is derived from the event name — e.g. “Heart Failure Forum 2026” → HFF2026. Receipts use the same pattern: HFF2026-REC-001.

ℹ️
Legacy events without an event code
If an event was created before the event code system, the code is automatically derived on the first invoice generation. No manual action needed.
Invoice types
Invoice
Issued before payment — the payment request sent to the attendee or their organisation.
Receipt
Issued after payment is confirmed via Stripe. Generated automatically on payment.
Credit Note
Issued for refunds or partial credits. Created alongside a refund transaction.
Viewing and tracking invoices
IMG-017
Invoice List
Invoices table — columns: invoice number, type (Invoice/Receipt/Credit Note), attendee name, amount, status (Draft/Sent/Paid/Overdue), issue date, due date, and PDF download button.
Chasing unpaid registrations

To send a payment reminder to all unpaid confirmed registrations, use the bulk email with payment filtering (Section 8.4). This targets only those who owe payment — paid and complimentary registrations are excluded automatically. One step, no manual filtering needed.

Refunds
🔴
Refunds must be processed via the dashboard — not via agents
Go to the registration → click Refund. This triggers the actual Stripe refund, updates payment status to REFUNDED, creates a credit note, and emails the registrant — all in one step. AI agents cannot initiate refunds. This is an intentional safety boundary.

Customising a template for your event

Sending a bulk email
IMG-019
Bulk Email Dialog
Send email dialog — recipient type toggle (Registrations / Speakers), registration status filter dropdown, payment status filter dropdown (visible for Registrations only), template selector, subject field, and recipient count preview before send.
Go to Communications → Send Email
Choose recipient type: Registrations or Speakers
Set Status filter (e.g. CONFIRMED only)
For registrations: optionally set Payment status filter (e.g. UNPAID only)
Select or compose the email → review recipient count → click Send
⚠️
Rate limit: 10 bulk sends per event per hour
Plan accordingly if sending multiple campaigns. Avoid sending back-to-back in quick succession.
Filtering recipients by status and payment
Example — Chase unpaid confirmed registrations
Recipient type: Registrations
Status filter: CONFIRMED
Payment filter: UNPAID

→ Email goes only to confirmed attendees who haven’t paid. Paid and complimentary registrants are excluded automatically.
Viewing and cancelling scheduled emails

Some emails are sent automatically by the system — webinar reminders, registration completion chasers. View the queue in Communications → Scheduled and cancel any you don’t want sent.

ℹ️
Manually scheduled campaigns are not available
Scheduled emails are system-generated based on event settings. All manually triggered bulk emails send immediately. If you need a timed campaign, contact your event coordinator to set up a system sequence.

Sponsorship & BD · Leadership
The organisation-wide contact database
IMG-020
Contacts List
Contacts table — columns: name, organisation, email, tags (shown as pills), and last event association. Search bar and tag filter at top.

Contacts is MM Group’s central contact database. Unlike registrations and speakers (event-specific), contacts are organisation-wide — a contact from one event remains searchable across all events. This is where your KOL database, association contacts, and institutional relationships live.

Adding and tagging contacts

When adding a contact, use tags to categorise them: e.g. cardiologist, kol, speaker, association-member. Tags make future searching and filtering much faster across all events.

Updating the sponsor list
🔴
Updating replaces the entire sponsor list

Known Limitations
What must be done manually or via the dashboard — not via AI agents.
All roles

These are intentional design boundaries or features not yet built. Your AI agents are aware of all of these and will direct you to the right alternative when they apply.

⛔
Ticket pricing — dashboard onlyTicket prices must be set and activated in Settings → Registration Types. AI agents cannot set pricing.
⛔
Refunds — dashboard + Stripe onlyRefunds must be initiated from the Registration Detail Sheet. AI agents cannot issue refunds. This is a deliberate safety boundary.
⛔
Media upload — dashboard onlyAll file uploads must be done through the Media Library. Agents can view and reference existing files but cannot upload new ones.
⛔
Sponsor list — replace-all, not mergeUpdating sponsors replaces the entire list. Always include all existing sponsors when adding a new one. See Section 10.
⛔
Scheduled emails — system-generated onlyYou cannot manually create a scheduled email campaign. All agent-triggered bulk emails send immediately.
⛔
Email address changes — dedicated flow requiredChanging a speaker’s or registrant’s email via the normal edit form is blocked. Use the dedicated Change Email process to prevent data inconsistencies.
⚠️
Remove speaker from session — under investigationThis function is returning an unexpected response in some cases. Use Replace All Speakers as the alternative until resolved. Your agent will handle this automatically.
